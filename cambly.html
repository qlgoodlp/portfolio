Data-Driven UA Optimization — Channel Diversification · Content Affinity Targeting · Behavioral Retargeting

Problem

Heavy reliance on Meta and Google (CAC rising +15% and +12% respectively) combined with broad targeting was causing structural deterioration in CAC and conversion efficiency.

Strategy

  • Channel Diversification: Introduced TikTok & Pinterest to create a 4-channel mix (Meta 35 / TikTok 25 / Google 25 / Pinterest 15)
  • Content Affinity Targeting: Segmented audiences by behavioral data (previously-read works, similar genres) to replace broad targeting
  • Behavioral Retargeting: Captured organic search intent outside the platform (Anime-related keywords) and connected via IP-matched retargeting
ROAS by Targeting Type — Behavioral Data Dominates

Result

  • Channel diversification reduced new user CAC by 32% YoY, breaking Meta/Google dependency
  • Content affinity targeting drove ROAS up to +229% (similar genre), pushing key action genre to revenue Top 3
  • Behavioral retargeting drove organic search volume ×10, social visitor ROAS MoM +132%
Key InsightCombining content attributes with users' past consumption & search intent — 'Content Affinity Targeting' — delivers dramatically higher ROAS than broad targeting.

Post-ATT iOS Measurement Response & First-Party Data Analytics Framework

Problem

After Apple's ATT rollout, user-level measurement for iOS campaigns became severely limited, making it difficult to accurately assess true user value from app campaign data alone.

Hypothesis

  • SKAN-based aggregated data has clear limitations — rebuilding the measurement framework around first-party User IDs is the only way to track long-term user value under ATT
Measurement Strategy Transition

Action

  • Expanded Web-to-App traffic share to compensate for iOS measurement limitations
  • Built a Snowflake-based User ID analysis environment using Singular integration data

Analysis

  • Connected the signup → purchase → repurchase path around first-party User IDs to track long-term user value
  • Segmented and analyzed purchase behavior by country and platform

Result

Secured a framework to analyze long-term user value using first-party data even under ATT, establishing the basis to shift KPI focus from short-term CPA to repurchase rate and ARPPU.

Key InsightATT's constraints are limitations of third-party tracking only. Treating first-party User IDs as an asset actually becomes an opportunity to evolve beyond short-term metrics toward an LTV-centric measurement system.

SQL-Based Funnel Bottleneck Diagnosis & Active User Maximization

Approach

Effective data analysis starts with understanding data structure (ERD). I first map how tables (user logs, payment logs, etc.) connect, then design analysis on top of that structure using SQL as the tool. This project shows how that capability led to a real funnel bottleneck diagnosis.

Problem

Analyzing the payment flow based on table relationships in the new-user-to-active-user funnel revealed that 'First Purchase' was the critical bottleneck.

Funnel Bottleneck Diagnosis

Strategy

  • Leveraging ERD understanding, directly extracted high-intent non-payer segments via SQL join queries across user and payment log tables
  • Deployed targeted CRM promotion: automated 50% cashback on first purchase
 high_intent_non_payers.sql
WITH user_activity AS (
  SELECT
    u.id AS user_id,
    COUNT(DISTINCT v.episode_id) AS viewed_episodes,
    COUNT(DISTINCT v.title_id) AS viewed_titles,
    SUM(CASE WHEN p.id IS NOT NULL THEN 1 ELSE 0 END) AS payment_cnt
  FROM users u
  LEFT JOIN view_log v ON u.id = v.user_id
    AND v.viewed_at >= NOW() - INTERVAL '30 days'
  LEFT JOIN payment_log p ON u.id = p.user_id
  GROUP BY u.id
)
SELECT user_id, viewed_episodes, viewed_titles
FROM user_activity
WHERE payment_cnt = 0  -- non-payers
HAVING viewed_episodes >= 20  -- high-engagement threshold
ORDER BY viewed_episodes DESC;

After mapping the relationships among users, view_log, and payment_log tables, this query uses a CTE to aggregate each user's 30-day episode views and payment status — directly surfacing the 'high-activity but non-paying' high-intent segment.

Result

Targeting high-intent non-payers improved DAU/MAU by 113%, and the targeted promotion pushed first purchase CVR up by 13 percentage points.

Core User Conversion Improvement via Cross-Genre Strategy

Problem

UA was focused on BL genre under the assumption that its low CAC would drive volume. However, a Snowflake SQL LTV cohort analysis of 17,819 paying users revealed that over 55% of BL-only users remained light users (LTV €30 or below), resulting in structurally low core user conversion rates.

Data Insight

Analyzing core user behavior (Minnow, LTV €100+) confirmed that 60% were cross-genre consumers (BL + Romance). Cross-genre users showed avg LTV +164%, active lifespan ×2.2, monthly spend +21%, and monthly episode purchases +47% vs. BL-only users.

Core User (Minnow+) Genre Consumption Pattern

Solution

  • CRM (In-app Push): Extracted BL reader segments from Snowflake and directly designed & ran personalized in-app push campaigns recommending top cross-genre pairs (e.g., Jinx → Hana's Demon of Lust)
  • Meta Paid Retargeting: Uploaded the same BL user segment as a Meta Custom Audience to serve Romance creatives, executing a multi-touch strategy linking CRM and paid channels
  • Cross-Pair Optimization: Validated high-converting BL→Romance pairings with quarterly data and updated priority exposure pairs

Result

  • Core user (Minnow+) ratio improved +15% QoQ
  • Cross-genre conversion users avg LTV +164%, median LTV +317%
  • Active lifespan extended ×2.2 — confirmed structural LTV improvement vs. CAC
Key InsightTesting the UA volume hypothesis (BL CAC is cheap) with data revealed that user quality (cross-genre experience) is the decisive variable for LTV. Snowflake cohort analysis → CRM + Paid multi-touch execution structurally improved core user ratio.
