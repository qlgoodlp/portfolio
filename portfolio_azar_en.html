Stage-Segmented CRM Messaging Map

Strategy

Segmented users into three stages by churn point and designed a tailored CRM messaging map for each stage's pain point.

Stage 1 · Churn before first free view
Pain PointFailure to find content of interest while browsing
Strategy: bestseller & interest-based curation
Stage 2 · Churn after free view, before first purchase
Pain PointInsufficient interest in the content
Strategy: recommend killer titles in similar genres
Stage 3 · Churn after first purchase, before repurchase
Pain PointResistance to additional payment after coins run out
Strategy: limited coin-purchase promotions & N% discounts

Result

Influencer activation lifted engagement 10x above average, and stage-tailored messaging delivered a first-purchase conversion rate 23% higher than other creatives.

Key InsightMarketing efficiency is maximized when the initial trust earned by influencers is handed off into precise, stage-specific CRM messages along the funnel.

SQL-based Funnel Bottleneck Diagnosis & Active-User Volume Maximization

Approach

My data analysis is grounded in an understanding of the data structure (ERD). I first map how tables — users, payment logs, and more — relate to one another, then use SQL as a tool to design analysis on top of that structure. This project is a case where that capability led directly to diagnosing a funnel bottleneck.

Problem

In the funnel converting newly acquired users into active users, analyzing the payment flow based on table relationships diagnosed 'First Purchase' as the critical bottleneck.

Funnel bottleneck diagnosis

Strategy

  • With an understanding of how the user and payment-log tables relate, extracted a high-engagement non-payer segment via SQL join queries
  • Ran a targeted CRM promotion: automated 50% cashback on first purchase
 high_intent_non_payers.sql
WITH user_activity AS (
  SELECT
    u.id AS user_id,
    COUNT(DISTINCT v.episode_id) AS viewed_episodes,
    COUNT(DISTINCT v.title_id) AS viewed_titles,
    SUM(CASE WHEN p.id IS NOT NULL THEN 1 ELSE 0 END) AS payment_cnt
  FROM users u
  LEFT JOIN view_log v ON u.id = v.user_id
    AND v.viewed_at >= NOW() - INTERVAL '30 days'
  LEFT JOIN payment_log p ON u.id = p.user_id
  GROUP BY u.id
)
SELECT user_id, viewed_episodes, viewed_titles
FROM user_activity
WHERE payment_cnt = 0  -- non-payers
HAVING viewed_episodes >= 20  -- high-engagement threshold
ORDER BY viewed_episodes DESC;

After mapping the relationships across the users, view_log, and payment_log tables, a CTE aggregates each user's 30-day viewing volume and payment status to directly derive the "high-engagement but non-paying" segment.

Result

Targeting high-engagement non-payers improved DAU/MAU by 113%, and the targeted promotion lifted first-purchase conversion rate (CVR) by 13%p.

Turning Existing Users Into an Acquisition Channel

Approach

Designed and ran social campaigns that use existing customers' own networks to reach new audiences — the core idea behind referral: happy customers become the most efficient acquisition channel.

  • Friend-tag campaign (Facebook / Instagram): participants shared their Halloween costume in the comments and tagged a friend to attend with — pulling each participant's friend network into brand exposure (UGC + invitation combined)
  • Share-to-reward event (X): fans followed and reposted with a support message to enter, extending organic reach through fan advocacy around an offline event and merchandise pop-up

Result

Expanded organic, earned reach by activating fans as advocates — bringing new audiences in through existing customers rather than paid media alone, while reinforcing engagement around live events.

Why it matters hereThe same instinct applies to an in-app service: a satisfied user who invites a friend is the highest-trust, lowest-cost acquisition there is — and friend-invite/share mechanics lift engagement and new sign-ups at the same time.

Adapting to Post-ATT iOS Measurement & Building a First-Party, Data-Driven Performance Analysis System

Problem

After Apple introduced ATT (App Tracking Transparency), user-level measurement of iOS campaigns became difficult, making it hard to accurately assess a user's true value from app campaign data alone.

Hypothesis

  • SKAN (SKAdNetwork) aggregate data alone has clear limits — the measurement system needs to be redesigned around first-party User IDs to keep tracking long-term user value under ATT
Measurement strategy shift

Action

  • Expanded the Web-to-App acquisition share to compensate for iOS measurement limits
  • Built a Snowflake-based User ID analysis environment on top of Singular-integrated data

Analysis

  • Connected signup → purchase → repurchase paths around first-party User IDs to track user-level long-term value
  • Segmented purchase behavior by country and platform

Result

Established a system that analyzes users' long-term value on first-party data even under ATT, and built the rationale to shift KPIs from short-term CPA to repurchase- and ARPPU-centric metrics.

Key InsightPlatform-imposed measurement constraints (ATT) are only a limit on third-party tracking — treating first-party User IDs as an asset turns the constraint into a catalyst for shifting toward long-term-value measurement beyond short-term metrics.

Data-Driven Performance UA — Media Diversification · Content-Affinity Targeting · Behavior-Data Retargeting

Problem

Dependence on Meta & Google (CPA +15% / +12%) plus simple broad targeting meant structurally worsening new-user acquisition costs (CPA) and conversion efficiency.

Strategy

  • Media diversification: launched TikTok & Pinterest, rebuilding into a 4-channel mix (Meta 35 / TikTok 25 / Google 25 / Pinterest 15)
  • Content-affinity targeting: replaced broad targeting with behavior-data audience segmentation (returning-season & similar-genre readers)
  • Behavior-data retargeting: captured external organic search intent (anime-related keywords) and connected it to IP-matched retargeting
ROAS by targeting approach — the edge of behavior data

Result

  • Media diversification cut new-user acquisition CPA 32% YoY, ending Meta/Google dependence
  • Content-affinity targeting lifted ROAS up to +229% (similar-genre), reaching Top 3 revenue in the key action genre
  • Behavior-data retargeting drove related-keyword organic search volume 10x and social-visitor ROAS +132% MoM
Key Insight"Content-affinity targeting" — combining content attributes with users' past consumption and search intent — delivers overwhelmingly higher ROAS than broad targeting.
